# AI辅助书籍生成项目 - 项目记忆

## 项目概述
- **项目名称**: AI辅助书籍生成器
- **版本**: v1.2.0
- **创建时间**: 2026-03-26
- **目标**: 将大文本文件（md/txt）通过豆包AI重新组织，生成结构完整的新书籍（DOC格式）

## 核心需求
1. 支持读取md和txt文件
2. 处理50万字级别的大文本
3. 持续性调用豆包API进行内容重组
4. 生成有目录、自序、详细章节的doc文档
5. 文风要求：平实朴素
6. 无语言逻辑错误
7. 最终字数与原文相近

## 技术栈
- **语言**: Python 3.8+
- **API**: 豆包 (Volces/Ark)
- **文档生成**: python-docx
- **配置格式**: YAML

## 处理流程
1. 文件读取 → 2. 内容分析 → 3. 大纲生成 → 4. 逐章生成 → 5. 审核优化 → 6. DOC输出

## 版本历史

### v1.2.0 (2026-03-26)
- **重大更新：多级章节结构**
- 实现三级目录结构：章（一级）→ 子章节（二级）→ 小节（三级）
- 每章包含3-4个子章节，每个子章节包含2-3个小节
- 大纲生成器重构，支持 `ChapterOutline` / `SubChapterOutline` / `SectionOutline` 三级数据结构
- DOC导出支持多级目录显示
- 终端输出显示完整层级结构

### v1.1.0 (2026-03-26)
- **重大更新：全文分批投喂功能**
- 新增 `full_text_analyzer.py` 模块，实现大文本分批投喂给AI
- AI现在会完整阅读50万字原文，而不是只看前5000字
- 修复章节数设置不生效的问题
- 新增 `logger.py` 日志系统模块
- 自序生成现在基于全文理解，不再乱写
- 优化终端输出，显示详细阅读进度

### v1.0.2 (2026-03-26)
- 修复相对导入问题
- 支持直接运行和模块导入两种方式

### v1.0.1 (2026-03-26)
